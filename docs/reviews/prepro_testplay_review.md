## 1. プランナーレビュー

### 良い点

1. **コアエンジンの完成度が極めて高い** — 65536分母抽選、5ライン判定、4コマ滑り制御、ゲーム数管理、RT、消灯/フラッシュ/遅れ/たまや振り分けテーブルが全て仕様書v0.3.0に準拠
2. **仕様書とコードの一貫性** — Flag enum、抽選テーブル数値、リール配列が一字一句一致
3. **デバッグインフラ整備** — debug_state.json、D/R/Tキーデバッグ、スクリーンショット保存
4. **アーキテクチャの健全性** — SlotEngine→シグナル→UIの疎結合。α以降でエンジン層の変更不要

### 指摘事項

| ID | 重要度 | 内容 |
|---|---|---|
| P1 | CRITICAL | テキストUIでは消灯・フラッシュ・遅れの「楽しさ」を評価不可。αで最優先解消 |
| P2 | HIGH | WAITING状態のawait中排他制御が脆弱。ボタン表示も未対応 |
| P3 | HIGH | 遅れ演出未実装（delay_fired/tamaya_firedシグナル未発行） |
| P4 | MEDIUM | ボーナス揃え引き込みが中段のみ。5ライン全てで判定すべき |
| P5 | MEDIUM | _judge_payout()にRPL×3の図柄判定なし（フラグベースのみ） |
| P6 | MEDIUM | _lottery_small_role_only()のテーブル直接erase問題 |
| P7 | LOW | セーブデータにゲーム状態（bonus/RT）が未保存 |

## 2. テクニカルリードレビュー

### 良い点

1. **アーキテクチャ分離が優秀** — シグナル経由の疎結合が完全維持
2. **型安全性** — Array[int]の明示型付け
3. **WaitTimerの設計** — 初回ウェイト回避、専用クラス分離
4. **セーブのバックアップ機構** — メイン→バックアップ→リカバリ
5. **CHERRY判定の優先処理** — 5ライン判定より先にLEFTリール位置依存の判定
6. **ボーナスストック中の再抽選差し替え** — 二重ストック防止

### 指摘事項

| ID | 重要度 | 内容 |
|---|---|---|
| T1 | **CRITICAL** | セーブデータにボーナス/RT状態が保存されない（仕様書11.1との大きな乖離） |
| T2 | **CRITICAL** | pull_lever()内のawait後に game_state 再確認がない（レースコンディション） |
| T3 | MAJOR | CHERRY_2/CHERRY_4のenum名が実払出と逆転（v0.3.0配当逆転時の名称未更新） |
| T4 | MAJOR | _judge_payout()にリプレイ(RPL)の図柄判定が欠落 |
| T5 | MAJOR | _lottery_small_role_only()でテーブルの直接改変 |
| T6 | MINOR | RT中ゲーム数デクリメントとボーナス当選時にrt_endedが2回発火の可能性 |
| T7 | MINOR | total_gamesのボーナス中カウント方針が仕様未明記 |
| T8 | MINOR | get_windowの上段/下段定義にコメントなし |
| T9 | MINOR | coin_historyが累積差枚数（グラフ用途なら正しいが意図を明記すべき） |

### α前に必須の修正

1. セーブデータへのボーナス/RT状態保存の実装
2. await後の状態再確認ガード追加
3. リプレイ図柄判定の追加

### α中に推奨

4. delay_fired / tamaya_fired シグナル発行処理の実装
5. CHERRY enum名の整理（CORNER_CHERRY / CENTER_CHERRY）
6. total_gamesのボーナス中カウント方針の仕様明確化
7. get_windowの上段/下段コメント追記
8. AudioManagerスタブの本実装

- L2 (通常): #EAEAFF — 本文テキスト
- L3 (重要): #00FF88 — 7セグ、有効LED
- L4 (注目): #FF1493 — アクセント
- L5 (特別): #FFD700 — BIG装飾、入賞

## 4. サウンドデザイナーレビュー

### サウンドコンセプト: 3層構造

| Layer | テーマ | 内容 |
|---|---|---|
| Layer 1 | 実機の質感 | レバー音・リール停止音・ウェイトチック音（メカニカルトランジェント） |
| Layer 2 | ネオンサイバー | デジタルの艶（ピッチライズ、フォルマント処理） |
| Layer 3 | VTuber/花火 | 感情ピーク演出（tamayaリバーブ、ファンファーレ） |

### αフェーズ優先SE/BGM

| 優先度 | SE/BGM | 方針 |
|---|---|---|
| 1 | lever_pull.wav | プロシージャル生成（金属音180Hz） |
| 2 | reel_stop_l/c/r.wav | プロシージャル生成（220/330/440Hz） |
| 3 | normal.ogg | 仮素材、ループポイント検証必須 |
| 4 | big_fanfare.wav | 仮ファンファーレ（3秒、Cメジャー） |
| 5 | wait_tick.wav | プロシージャル生成（1000Hz/50ms） |
| 6 | medal_out.wav | ループ再生+停止ロジック検証 |

### AudioManager設計提案

```gdscript
func play_se(name: String, volume_db: float = 0.0) -> void
func play_bgm(name: String, fade_sec: float = 0.5) -> void
func duck_bgm(target_db: float, fade_sec: float, restore_after: float) -> void
func stop_se(name: String) -> void  # medal_outループ停止用
```

### 技術アイデア

- **プロシージャル生成**: sound_generator.gd で音源ファイルなしでα音響レビュー可能
- **RTフィルタースウィープ**: AudioEffectFilter + cutoff_hz リアルタイム制御
- **配信スタジオ感**: EQで8kHz以上-3dB、250Hz+2dB（オプション、β以降）
- **遅れ演出**: lever_pull のRelease 0.2-0.3秒で「不気味な間」を演出
