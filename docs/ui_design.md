## 3. スマートフォン視認性・操作性基準

本セクションは「実際の人間がスマートフォンで不足なく認識・操作できること」を保証するための定量的基準である。
全てのUI要素はこの基準を満たさなければならない。

### 3.1 対象デバイスと換算基準

| デバイスクラス | 画面幅 (px) | DPI | スケール係数 | 1 vp-px (mm) | 想定機種例 |
|---|---|---|---|---|---|
| **Budget（最小保証）** | 720 | 320 | 0.80x | 0.063 | Galaxy A14, Redmi 12C |
| Standard | 1080 | 400 | 1.20x | 0.076 | Pixel 7a, Galaxy A54 |
| High-end | 1440 | 560 | 1.60x | 0.073 | Galaxy S24 Ultra |

- Viewport: 900x1600、Stretch: canvas_items / expand
- スケール係数 = デバイス画面幅 / 900
- **全ての基準は Budget デバイス（0.80x スケール、320dpi）で検証する**
- Budget で合格すれば Standard / High-end でも自動的に合格する

### 3.2 テキスト視認性基準

#### 最小フォントサイズ

| 区分 | 最小 viewport px | Budget 物理高さ | 根拠 |
|---|---|---|---|
| **プレイヤー向けテキスト** | **24px** | 1.51mm | Apple HIG 最小 11pt (≈1.5mm) 準拠 |
| **補助テキスト**（ヘルプ、キャプション） | **22px** | 1.39mm | Material Design caption 最小相当 |
| **デバッグ専用**（プレイヤー非表示） | 18px | 1.13mm | 開発者のみ。リリースビルドでは非表示 |

**ルール**:
- プレイヤーが読む必要のあるテキストは **22px viewport 未満を禁止**
- ゲーム進行に必要な情報（CREDIT、ゲーム数、ボタンラベル等）は **24px 以上**
- 18px はデバッグオーバーレイ限定。リリースビルドでは表示しない

#### 行間・文字間隔

| 項目 | 基準値 | 根拠 |
|---|---|---|
| 行間（line-height） | フォントサイズの **1.3倍以上** | WCAG 1.4.12 テキスト間隔 |
| 文字間隔 | デフォルト以上（詰めない） | 小画面での可読性維持 |
| テキスト周囲余白 | フォントサイズの **0.5倍以上** | 背景との分離 |

#### テキスト切れ防止

- 数値表示（CREDIT等）は最大桁数（9999）で表示テスト済みであること
- 日本語テキストは最長想定文字列で折り返し・切れがないこと
- `clip_text = true` を使う場合、切れる状況が発生しないことを確認

### 3.3 タッチ操作性基準

#### タッチターゲットサイズ

| 区分 | 最小 viewport px | Budget 物理サイズ | dp換算 (320dpi) | 根拠 |
|---|---|---|---|---|
| **主操作ボタン**（BET/LEVER/STOP） | **120 x 120** | 9.6 x 9.6mm | 48dp | Material Design 推奨最小 |
| 副操作ボタン（設定、戻る等） | **90 x 90** | 7.2 x 7.2mm | 36dp | Material Design 絶対最小 |
| 情報表示のみ（タップ不可） | — | — | — | タッチ判定なし |

**ルール**:
- ゲーム進行中に頻繁にタップするボタン（BET/LEVER/STOP）は **120x120px viewport 以上**
- 設定画面等の低頻度ボタンでも **90x90px viewport 以上**
- アイコンが小さくてもタッチ判定領域（CollisionShape / Button rect）は上記サイズを確保

#### ボタン間隔（誤タップ防止）

| 項目 | 最小 viewport px | Budget 物理 | 根拠 |
|---|---|---|---|
| 隣接ボタン間の間隔 | **20px** | 1.6mm | Material Design 8dp gap |
| 画面端からの余白 | **30px** | 2.4mm | エッジ誤タップ防止 |

**ルール**:
- 隣接するタップ可能要素の間に最低 20px のスペースを確保する
- 画面左右端から 30px 以内にタップ可能要素を配置しない

#### 現行ボタンサイズの適合状況

| ボタン | 現行サイズ | 基準 | 判定 |
|---|---|---|---|
| BET | 80x120 | 120x120 | **NG — 幅不足（80→120に拡大必要）** |
| LEVER | 100x120 | 120x120 | **NG — 幅不足（100→120に拡大必要）** |
| STOP L | 150x120 | 120x120 | OK |
| STOP C | 150x120 | 120x120 | OK |
| STOP R | 150x120 | 120x120 | OK |

全5ボタン合計幅: 120+120+150*3 = 690px（900px に対して余裕あり）

### 3.4 カラーコントラスト基準

WCAG 2.1 AA 準拠を必須とする。

| テキスト区分 | 最小コントラスト比 | 根拠 |
|---|---|---|
| **通常テキスト**（< 24px viewport） | **4.5:1** | WCAG 2.1 AA |
| **大テキスト**（≥ 24px viewport Bold / ≥ 28px Regular） | **3.0:1** | WCAG 2.1 AA Large Text |
| **非テキスト UI 要素**（ボタン枠線、アイコン等） | **3.0:1** | WCAG 2.1 AA Non-text |

#### カラーパレット適合検証

背景 #0A0A1E に対する各色のコントラスト比:

| 色 | HEX | コントラスト比 | AA通常 (4.5:1) | AA大 (3.0:1) |
|---|---|---|---|---|
| White | #EBEBFF | 16.8:1 | PASS | PASS |
| Neon Green | #00FF88 | 14.8:1 | PASS | PASS |
| Gold | #FFD700 | 14.1:1 | PASS | PASS |
| Neon Cyan | #00D4FF | 11.1:1 | PASS | PASS |
| Muted Gray | **#8888A0** | 5.7:1 | PASS | PASS |
| Neon Pink | #FF1493 | 5.4:1 | PASS | PASS |
| Purple | #B14EFF | 5.1:1 | PASS | PASS |
| Blue | #2979FF | 4.9:1 | PASS | PASS |
| Red | #FF1744 | 4.6:1 | PASS | PASS |

**注意**: 旧 Muted Gray #666680 はコントラスト比 3.6:1 で通常テキスト不適合のため **#8888A0** に変更済み。

#### パネル背景 #1A1A2E に対する検証

#1A1A2E 上でテキストを表示する場合も同一基準を適用する。
暗色パネル上に暗色テキストを配置しないこと。

### 3.5 図柄・アイコン判別基準

| 項目 | 基準値 | 根拠 |
|---|---|---|
| リール図柄最小サイズ | 120x96px viewport (7.6x6.1mm Budget) | 現行 200x160px で十分 |
| 小アイコン最小サイズ | 32x32px viewport (2.0x2.0mm Budget) | Material Design icon touch 補助 |
| 隣接図柄の色相差 | 30度以上（HSV色相環） | 色覚多様性を考慮した最低基準 |

**リール図柄の判別可能性**:
- 7種の図柄は全て異なる主色相をもつ（赤/青/黄/ピンク/金/シアン/緑）
- 色だけでなく形状（数字/文字/アイコン形状）でも区別可能であること
- 色覚特性に依存しない判別手段（形状、テクスチャ、テキスト）を最低1つ持つこと

### 3.6 情報密度基準

| 項目 | 基準値 | 根拠 |
|---|---|---|
| 1ゾーン内の情報項目数 | **最大5項目** | 認知負荷軽減（Miller's Law 7±2 の下限） |
| テキスト行数/ゾーン | ゾーン高さ ÷ (フォントサイズ × 1.5) 以下 | 行間確保 |
| ゾーン間の視覚的区切り | 色差、罫線、余白のいずれか | 領域の認知的分離 |

**ルール**:
- 1画面に同時表示する情報は、プレイヤーが2秒以内に把握可能な量に制限する
- 常時表示情報（CREDIT/ゲーム数）と一時表示情報（WIN演出/リーチ目告知）を分離する
- 一時表示情報は自動的に消える（表示時間 2-5秒）か、次のアクションで消えること

### 3.7 検証チェックリスト

新画面作成時・UIレビュー時に以下を全項目確認する。

#### テキスト
- [ ] プレイヤー向けテキストが全て 22px viewport 以上か
- [ ] ゲーム進行に必要な数値・ラベルが全て 24px viewport 以上か
- [ ] 18px テキストがデバッグ用途のみで、リリースビルドで非表示か
- [ ] 最大桁数・最長文字列で切れ・はみ出しがないか
- [ ] 行間がフォントサイズの 1.3倍以上か

#### タッチ
- [ ] 主操作ボタンが全て 120x120px viewport 以上か
- [ ] 副操作ボタンが全て 90x90px viewport 以上か
- [ ] 隣接ボタン間に 20px 以上のスペースがあるか
- [ ] 画面端 30px 以内にタップ可能要素がないか

#### コントラスト
- [ ] 全テキストが背景に対して WCAG AA (4.5:1) 以上か
- [ ] 大テキスト（24px Bold / 28px以上）は 3.0:1 以上か
- [ ] ボタン枠線・アイコンが背景に対して 3.0:1 以上か
- [ ] #666680 が使われていないか（#8888A0 に置換済みか）

#### 図柄
- [ ] 全図柄が色と形状の両方で判別可能か
- [ ] 小アイコンが 32x32px viewport 以上か

#### レイアウト
- [ ] 1ゾーン内の情報項目が 5項目以下か
- [ ] 常時表示と一時表示が視覚的に分離されているか
- [ ] ゾーン間に視覚的区切り（色差/罫線/余白）があるか
